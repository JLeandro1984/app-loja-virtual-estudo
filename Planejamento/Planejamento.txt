Acho que vale a pena estruturar isso como um **projeto profissional**, semelhante ao planejamento de um sistema corporativo. Em vez de começar codificando, definimos primeiro a arquitetura, os padrões e o roadmap. Isso reduz retrabalho e mantém o projeto consistente durante toda a evolução.

---

# Planejamento Geral do Projeto

**Nome do Projeto:** Catálogo Digital Premium

**Versão Inicial:** MVP 1.0

**Objetivo:** Desenvolver um catálogo digital premium utilizando HTML5, CSS3 e JavaScript puro, preparado para evoluir futuramente para uma plataforma completa de vendas com backend, banco de dados e painel administrativo.

---

# Visão Geral

## Objetivos Principais

### MVP

Desenvolver um catálogo premium que permita:

* Exibir produtos.
* Organizar produtos por categorias.
* Pesquisar produtos.
* Filtrar produtos.
* Ordenar produtos.
* Exibir detalhes completos.
* Selecionar produtos.
* Gerenciar um carrinho de interesse.
* Enviar orçamento via WhatsApp.
* Direcionar para Instagram.
* Compartilhar produtos.
* Funcionar perfeitamente em desktop e dispositivos móveis.

# Critérios de Qualidade

Cada fase somente poderá ser considerada concluída quando atender aos seguintes critérios:

* Arquitetura consistente e documentada.
* Código modular e reutilizável.
* Separação clara de responsabilidades.
* HTML semântico.
* CSS organizado por componentes.
* JavaScript em módulos ES.
* Responsividade completa.
* Compatibilidade com navegadores modernos.
* Performance otimizada.
* Acessibilidade seguindo boas práticas.
* Preparação para futura integração com APIs.
* Ausência de código duplicado.
* Ausência de dependências desnecessárias.
* Organização adequada para manutenção e evolução.

## Minha recomendação final

Em vez de produzir apenas um grande prompt, eu transformaria este projeto em uma **documentação de engenharia de software** organizada em múltiplos documentos (`/docs`), por exemplo:

* `01-visao-geral.md`
* `02-requisitos-funcionais.md`
* `03-arquitetura.md`
* `04-modelo-de-dados.md`
* `05-design-system.md`
* `06-plano-de-desenvolvimento.md`
* `07-padroes-de-codigo.md`
* `08-roadmap.md`
* `09-criterios-de-qualidade.md`
* `10-prompt-mestre.md`

Essa organização facilita a manutenção, permite evoluir cada documento de forma independente e serve como referência durante todo o ciclo de vida do projeto. Além disso, o `10-prompt-mestre.md` pode consolidar e referenciar os demais documentos, orientando qualquer IA a seguir a arquitetura definida em vez de gerar uma solução isolada.
